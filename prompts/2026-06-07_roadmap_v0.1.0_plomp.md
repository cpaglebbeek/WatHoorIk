# Roadmap — v0.1.0-Plomp: klinische centrale-auditieve module

## Aanleiding

Cross-repo trigger uit `GezondheidJoyce/dossiers/sensorische_blokkade/` (zie `prompts/2026-06-07_newp_klinisch_sensorische_blokkade.md` aldaar). Voor empirisch onderzoek naar centrale auditieve verwerking is een **generieke** zelf-test-batterij T1-T6 nodig die niet aan één persoon gebonden is — past in WatHoorIk als nieuwe tab "Klinisch" (publiek bruikbaar voor iedereen met "ik hoor maar versta niet"-klacht).

## Volgorde

**Eerst:** v0.0.8-Pollack (bestaande roadmap, A-blok: live verbinding tester/kandidaat, eerste backend `api.php` op Hostinger). Pollack is een functionele afhankelijkheid voor Plomp's tester-modus (Christian = tester, Joyce = kandidaat met minimale UI).

**Daarna:** v0.1.0-Plomp.

## Codenaam Plomp

**Reinier Plomp** (1928-2016, NL) — pionier speech-in-noise audiometrie, ontwikkelde Plomp/Mimpen-zinnen (SRT). Past in cochlea-/audiometrie-pioniers-thema. Major version-bump (.1.0) want oranje feature-impact: nieuwe tab, nieuwe localStorage-namespace, nieuwe JSON-export-schema.

## Scope v0.1.0-Plomp

Nieuwe tab "Klinisch" met zes sub-tests:

| Test | Beschrijving | Web Audio-techniek |
|------|-------------|-------------------|
| **T1 SIN** | NL-zinnen + 4-talker babble; adaptief SNR via 1up/1down regel; SRT50% rapport | AudioBuffer multitrack mix + BiquadFilter normalize + adaptive level via GainNode |
| **T2 GIN** | wit ruis 6s met 0-10 silent gaps random positie 2-20 ms; user-press detectie | OfflineAudioContext + AudioBufferSourceNode (wit ruis) + envelope-gating per gap |
| **T3 Dichotic Digits** | 2 verschillende cijfers L+R simultaan; herhaal beide; rapporteer % per oor | 2× OscillatorNode + StereoPannerNode pan ±1 + spraak-samples via AudioBuffer |
| **T4 Env-vs-Words** | 20 omgevingsgeluiden (klop, fluit, hond, klok...) + 20 NL woorden bij identieke SNR | preloaded AudioBuffers + noise mix |
| **T5 Rep+Write** | zin gepresenteerd → user spreekt na (MediaRecorder) én typt; daarna geschreven zin → user spreekt na | MediaRecorder + textarea + WAV-encoder hergebruik uit v0.0.2 |
| **T6 ACPT** | 10 min stream NL-woorden met target-woord; spatie-press; meet hits/FA/RT-drift | scheduled AudioBufferSourceNode + performance.now() RT |

**Resultaten:**
- localStorage key `wathoorik_klinisch_v1`
- JSON-export schema `wathoorik-clinical-v1`
- PDF-export via jsPDF (hergebruik patroon Hood)
- Per test: score + norm-band + indicatie

**Tester-modus (na Pollack):**
- Tester (op laptop) ziet alle resultaten realtime.
- Kandidaat (op telefoon) ziet alleen "Ik hoor het" / "Ik begrijp het" / "Ik val weg"-knoppen.
- HTTP polling 600ms (zelfde patroon als Pollack).

## Disclaimers (verplicht in UI bij Plomp)

- Indicatief, geen medisch hulpmiddel.
- Bij positieve scores: verwijs naar audioloog of huisarts.
- Resultaat-afhankelijk van hoofdtelefoon-type, omgevingsruis, systeem-volume.
- Geen koppeling met patiëntendossier (privacy by design — WatHoorIk is client-side).
